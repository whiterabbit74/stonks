# Аудит мобильной версии сайта

Ниже представлен подробный список выявленных проблем дизайна и верстки при анализе мобильной версии приложения.

## 1. Критические проблемы верстки (Overflow & CSS Conflicts)

### 1.1. Глобальный стиль минимального размера кликабельных элементов
В файле `src/index.css` задан глобальный стиль для всех кнопок и инпутов:
```css
button, [role="button"], ... {
  min-height: 44px;
  min-width: 44px;
}
```
**Проблема:** Это правило конфликтует с компактными UI-элементами, использующими утилитарные классы Tailwind (например, `h-5`, `w-8`).
**Пример:** Компонент `ErrorLogButton.tsx` использует `h-5` (20px), но из-за глобального стиля он будет растянут до 44px, что сломает верстку или приведет к визуальным артефактам. То же самое касается кнопок в `MiniQuoteChart` или тулбарах.

### 1.2. Фиксированная минимальная ширина таблиц
В `src/components/TradeDurationChart.tsx`:
```tsx
<table className="min-w-[360px] text-sm">
```
**Проблема:** Ширина 360px превышает ширину экрана старых устройств (iPhone SE 1-го поколения, некоторые Android имеют 320px). Даже на iPhone 12/13 mini (375px) с учетом отступов (padding) это вызовет горизонтальный скролл всего контейнера или обрезание контента.

### 1.3. Агрессивная минимальная ширина инпутов
В `src/components/BuyAtClose4Simulator.tsx`:
```tsx
className="flex-1 min-w-[300px] ..."
```
**Проблема:** На экранах 320px (и даже 375px с отступами) этот инпут займет почти все пространство, ломая flex-контейнер, если рядом есть другие элементы, или вызывая перенос строк там, где это не задумано.

## 2. Проблемы UX и адаптивности (Usability)

### 2.1. Избыточная высота графиков
В `src/components/Results.tsx`:
```tsx
<div className="h-[65vh] min-h-[500px] max-h-[900px] ...">
```
**Проблема:** `min-h-[500px]` — это слишком много для мобильного экрана. На iPhone 12 (высота viewport ~750-800px, минус браузерные панели ~650px) график займет почти весь экран, не оставляя места для заголовка, навигации или контекста. Пользователь не поймет, где он находится, пока не проскроллит.

### 2.2. Навигация BottomNav
В `src/components/ui/BottomNav.tsx`:
- Индикаторы страниц (точки) позиционированы абсолютно: `absolute -top-3`.
**Проблема:** Они могут перекрывать контент в самом низу страницы (например, кнопку "Load More" или футер), либо быть визуально оторванными от навигации.
- Использование свайпов (`onTouchStart`...) на всей навигации может конфликтовать с системными жестами (например, на iOS swipe to go back), если зона захвата слишком велика.

### 2.3. Таблицы данных
В `TradesTable.tsx` (и других таблицах) не всегда явно задан контейнер с `overflow-x-auto`. Если таблица имеет много колонок, она растянет страницу по ширине, вызывая горизонтальный скролл всей страницы (body), что является грубой ошибкой мобильного UX. (Требует проверки наличия обертки `.horizontal-scroll` вокруг всех таблиц).

### 2.4. Модальные окна
`InfoModal.tsx` и другие модальные окна должны проверяться на наличие `max-h-[80vh]` и `overflow-y-auto`, иначе на маленьких экранах контент модального окна может уйти за пределы экрана без возможности прокрутки.

## 3. Рекомендации

1.  Заменить глобальный стиль `min-height: 44px` на утилитарный класс `.touch-target`, применяемый только к основным кнопкам, либо использовать `:not(.compact)` селектор.
2.  Заменить жесткие `min-w-[360px]` на `w-full min-w-0` или использовать горизонтальный скролл для таблиц.
3.  Уменьшить `min-h` для графиков на мобильных устройствах (использовать `md:min-h-[500px]`, а для мобильных — `h-[40vh]` или `min-h-[300px]`).
4.  Проверить все инпуты на `w-full` вместо фиксированных ширин.
